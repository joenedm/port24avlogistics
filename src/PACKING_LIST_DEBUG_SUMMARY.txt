                        PACKING LIST BUG FIX SUMMARY
═══════════════════════════════════════════════════════════════════════════════

WHAT FILE WAS WRONG?
────────────────────
pages/Scan — The Scan In/Out page with Session Mode (packing/bulk scanning)

WHAT DATA SOURCE WAS INCORRECTLY USED?
──────────────────────────────────────
1. Assets were being cached indefinitely (staleTime not set)
2. No real-time subscription meant changes weren't detected
3. No refresh on tab/window focus meant stale data persisted
4. Sub-location filtering was missing from the packing list

WHAT FILTER LOGIC WAS CHANGED?
───────────────────────────────
BEFORE:
  showAssets = assets.filter(a => a.current_show_id === selectedShowId)
  (This was correct, but assets were stale)

AFTER:
  1. Real-time subscription invalidates cache on any asset change
  2. Assets query marked as stale (staleTime: 0)
  3. Refetch on window focus enabled
  4. Sub-location filtering added:
     if (selectedSubLocationId) {
       showAssets = showAssets.filter(a => a.current_sub_location_id === selectedSubLocationId)
     }
  5. Debug logs added to show filtering at each stage

SPECIFIC FIXES APPLIED:
───────────────────────
✓ Added Asset.subscribe() to invalidate cache on changes
✓ Set staleTime: 0 to prevent cache persistence
✓ Added refetchOnWindowFocus: true for tab switching
✓ Enhanced filter to include optional sub-location
✓ Added detailed console.log() for debugging
✓ Expanded filterlogic to show 3 stages: DB total → assigned to show → assigned to sublocation
